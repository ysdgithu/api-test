# API 接口测试技能

## 简介

这是一个API 接口测试技能，支持自动化接口测试流程。适用于存量项目的接口测试，包括新项目/模块测试、新增接口测试、修改接口测试及回归测试。

## 功能特性

- **自动探测项目信息**：自动识别项目类型、认证方式、API 基础地址等
- **测试用例生成**：自动生成结构化的测试用例文档
- **自动化测试执行**：创建并执行测试脚本
- **测试报告生成**：生成详细的测试报告，包含失败详情分析
- **支持多种认证方式**：JWT、Token、Basic Auth、无认证等

## 使用方法

### 触发关键词

- 阶段1（新建测试）：`接口测试`、`API测试`、`接口联调`
- 阶段2（确认执行）：`测试用例没问题`、`可以继续`、`确认执行`
- 重新运行：`重新跑测试`、`运行测试`、`执行测试`

### 工作流程

#### 首次测试

1. 触发测试（例如：`帮我测试 posts 模块的接口`）
2. AI 自动探测项目信息并展示给你确认
3. 补充必要信息（测试账号、测试接口等）
4. AI 生成测试用例文档（`test_cases.md`）
5. 确认后执行测试
6. 生成测试报告（`test_report.md`）

#### 重新测试

直接运行脚本文件即可。

## 生成的文件

测试相关文件存储在项目模块目录下：

```
{project_root}/{module_name}/
├── test_cases.md          # 测试用例文档（供审核）
├── test_script.py         # 测试脚本（可执行）
└── test_report.md         # 测试报告（输出）
```

## 注意事项

1. 测试文件需要**手动**添加到 `.gitignore`
2. 生成测试用例后需要手动确认才会执行测试

## 依赖

- Python 3.x
- requests 库（用于 HTTP 请求）

## 示例

```
用户: 帮我测试 posts 模块的接口

AI: 已自动探测到以下项目信息：
    项目根目录: /path/to/project
    项目地址: http://localhost:8000
    认证方式: JWT
    登录接口: /api/token/

    请核对以上信息，并补充：
    1. 是否需要创建测试账号？
    2. 需要测试的接口列表
```
